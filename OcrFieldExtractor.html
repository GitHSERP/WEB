OcrFieldExtractor 操作說明

OcrFieldExtractor 是建立在 OneOcrClient 之上的「欄位抽取」工具單元。流程為:載入影像 → OCR → 在指定區域內找「標籤(label)」→ 沿指定方向(右/左/上/下)取最近的「值(value)」。適合報單、發票、申請書等「LABEL  VALUE」式的制式表單。

API 採兩步驟設計:FindLabel 只找標籤,FindValueInDirection 再依方向取值。職責分離,呼叫端決定方向(fdRight / fdLeft / fdUp / fdDown),也方便對同一個 label 試多個方向。需要把同一直欄的值連續往下抓多個時,用 FindNextInDirection 以「上一個值」為新錨點繼續找。若 label 被 OCR 拆成同列多個 box(例:報單號碼 → 報單 + 號碼),用 FindKeywordBoxes 以 Rows 重組關鍵字、取得組成它的 box 陣列。

一、前置準備

  1. 將 OcrFieldExtractor.pas 連同其相依 OneOcrClient.pas、WinJson.pas 加入專案。
  2. 確認 xAISDK1.exe 可被 TOneOcrClient 找到(主程式 EXE 同目錄,或於 OneOcrClient 中設定 ExePath)。詳見 OneOcrClient.html。
  3. 在 uses 區塊加入 OneOcrClient, OcrFieldExtractor。
uses
  OneOcrClient, OcrFieldExtractor;

二、快速開始

最常見用法:LoadImage 一次,後續多次 FindLabel + FindValueInDirection 共用同一份 OCR 結果。

var
  Ex:       TOcrFieldExtractor;
  Box:      TFieldBox;   // 從 FindLabel 拿到的「只含 label」的 Box
  ValueBox: TFieldBox;   // 從 FindValueInDirection 拿到的「label + value」
begin
  Ex := TOcrFieldExtractor.Create;
  try
    Ex.PaddingPixels := 25;             // 視需要;0 表關閉
    if not Ex.LoadImage('C:\1.jpg') then
      raise Exception.Create(Ex.LastError);

    // 整張圖找標籤,再沿右側取值
    Box := Ex.FindLabel('報單號碼');
    if Box.Found then
    begin
      ValueBox := Ex.FindValueInDirection(Box, fdRight);
      if ValueBox.Found then ShowMessage(ValueBox.Value);
    end;

    // 若整張圖會撞到同名標籤,改用實際區域
    Box := Ex.FindLabel('匯率', TRegionRect.Make(L, T, R, B));
    if Box.Found then
    begin
      ValueBox := Ex.FindValueInDirection(Box, fdRight);
      if ValueBox.Found then ShowMessage(ValueBox.Value);
    end;

    // 同一個 Box 可以試多方向 — FindLabel 只跑一次,Box 不要被覆寫
    Box := Ex.FindLabel('總計');
    if Box.Found then
    begin
      ValueBox := Ex.FindValueInDirection(Box, fdRight);
      if not ValueBox.Found then
        ValueBox := Ex.FindValueInDirection(Box, fdDown);   // 右側沒值,改試下方
      if ValueBox.Found then ShowMessage(ValueBox.Value);
    end;

    // 需要格式驗證 → 呼叫端自行套 TRegEx.IsMatch(ValueBox.Value, '...')
  finally
    Ex.Free;
  end;
end;
Found 代表「該次呼叫整體成功」。FindLabel 回傳的 Box,Found 就是「label 是否找到」;FindValueInDirection 回傳的 Box,Found 是「label 找到且該方向有取到值」。需要區分「label 沒找到」vs「label 找到但無值」時,看 LabelLineIdx >= 0。
區域座標怎麼來?在 OneOcrFormDemo 主表單按下 Extract Fields,可列印出每條 OCR 行的 bbox 對照表,直接抄回程式碼。

三、比對演算法

整個流程可拆成四步:

  1. OCR 一次,後續共用:LoadImage 內部呼叫 TOneOcrClient.Recognize,把 TOcrFileResult 快取於 OcrResult 屬性。之後任意次 FindLabel / FindValueInDirection 都只在記憶體中比對。
  2. 找標籤:遍歷 OcrResult.Lines,以 Quad 中心點是否落入 TRegionRect 判定區域命中,再依 TLabelMatchMode 比對文字。多個命中時取「最上、最左」者。回傳的 TFieldBox 只填 Label*,Value* 留空。
  3. 依方向收集候選:呼叫 FindValueInDirection(Box, ADir) 後,以 label 為基準依 ADir 收集候選:
    • fdRight / fdLeft:候選必須同列(兩 Quad 的 Y 區間重疊比例 ≥ RowTolerance,預設 0.5);方向以「候選緣 vs label 緣 + 5% slack」判定,距離沿 X 軸主軸邊到邊量測。
    • fdUp:候選必須同欄(X 區間重疊比例 ≥ RowTolerance);方向以「候選中心 Y < label 中心 Y」判定(避免 OCR 框上下緣略重疊時被排除),距離沿 Y 軸主軸邊到邊量測。
    • fdDown:候選必須同欄(X 區間重疊比例 ≥ RowTolerance);方向以「候選中心 Y > label 中心 Y」判定,排序鍵改用中心 dy(候選中心 Y − label 中心 Y),適合欄高不齊或值偏離 label 中線時穩定取最近列。
    • 距離超過「label 另一軸尺寸 × 50」視為跨欄/跨列,不認列(常數 MAX_DISTANCE_FACTOR = 50.0)。
  4. 選擇值:候選依排序鍵由近到遠排序(fdRight/fdLeft/fdUp 用主軸邊到邊距離,fdDown 用中心 dy),取最近者填入 Value* 並把 Found 設為 True;無候選時 Value=''、ValueLineIdx=-1、Found=False(LabelLineIdx 仍保留以便除錯或試別的方向)。

四、主要 API

LoadImage

function LoadImage(const AImagePath: string;
  ACreateAnnotated: Boolean = False): Boolean;

載入影像並執行 OCR。成功回 True 並把結果寫入 OcrResult;失敗回 False,訊息見 LastError。重複呼叫會覆蓋上次的結果。ACreateAnnotated=True(除錯模式)會在來源圖旁產生標註版 JPG,同時保留 OCR 原始 <image>_J.txt(座標已還原回原圖),方便對照人工檢查。

FindLabel(整張圖)

function FindLabel(const ALabel: string;
  AMatchMode: TLabelMatchMode = lmStripParens): TFieldBox; overload;

在整張圖中找 label。回傳的 TFieldBox 只填 LabelText / LabelQuad / LabelLineIdx;Value* 留空,呼叫端再以 FindValueInDirection 取值。等同 FindLabel(ALabel, TRegionRect.Whole, AMatchMode)。

FindLabel(指定區域)

function FindLabel(const ALabel: string;
  const ARegion: TRegionRect;
  AMatchMode: TLabelMatchMode = lmStripParens): TFieldBox; overload;

限制 label 中心點必須落在 ARegion 之內。當整張圖會撞到相同 label(例如表單上下半都有「金額」)時,改用此重載。

FindValueInDirection

function FindValueInDirection(ABox: TFieldBox;
  ADir: TFieldDir): TFieldBox;

沿指定方向取 ABox 標籤的最近值。回傳全新獨立的 TFieldBox(不會修改 ABox):

  • Label* 從 ABox 搬入。
  • Value* 填該方向的值;找不到時 Value=''、ValueLineIdx=-1。
  • Found 預設 False;ABox 有效(ABox.Found 為 True 且 LabelLineIdx>=0)且該方向真有取到值時才設為 True。

FindNextInDirection

function FindNextInDirection(ABox: TFieldBox;
  ADir: TFieldDir): TFieldBox;

以 ABox 的「值」(ValueQuad / ValueLineIdx)為新錨點,沿 ADir 繼續找下一個。語意是「從上一個值再走一步」,適合做表格垂直串接,例如把同一欄的明細逐列抓出來。

注意:若改用 FindValueInDirection(prevResult, fdDown) 重複呼叫,因為 LabelLineIdx 沿用原 label,會一直回到同一個值——所以必須用 FindNextInDirection 才能往下推進。

DumpCandidatesInDirection

function DumpCandidatesInDirection(ABox: TFieldBox; ADir: TFieldDir;
  AUseValueAsAnchor: Boolean = False): string;

除錯工具。列出 ABox 在 ADir 方向上「通過所有篩選(對齊門檻、方向、MaxDist)的候選」,並印出每個候選的 AxisDist(主軸邊到邊距離)、dx/dy(中心相對位移)、Overlap(同列或同欄重疊比例)與原文。第 1 列以 * 標記,就是 FindValueInDirection / FindNextInDirection 實際會選中的那一條。

  • AUseValueAsAnchor=False:以 ABox.LabelLineIdx 為錨,等同診斷 FindValueInDirection(ABox, ADir)。
  • AUseValueAsAnchor=True:以 ABox.ValueLineIdx 為錨,等同診斷 FindNextInDirection(ABox, ADir)。

排序鍵與實際 Find* 一致(fdDown 用中心 dy,其餘用主軸邊到邊距離)。回傳是多行字串,直接 ShowMessage 或塞進 TMemo 即可閱讀。典型用法:當某個欄位串抓到非預期的值,先呼叫此函式看候選排行,即可判斷是「真的值沒通過篩選」還是「另一個候選的排序鍵更小」。

FindKeywordBoxes

function FindKeywordBoxes(const AKeyword: string;
  AMatchMode: TLabelMatchMode = lmContains): TKeywordMatch; overload;

function FindKeywordBoxes(const AKeyword: string;
  const ARegion: TRegionRect;
  AMatchMode: TLabelMatchMode = lmContains): TKeywordMatch; overload;

找出「拼出關鍵字字串的 box 陣列」。專門處理 OCR 把一個 label 拆成同一列多個 box 的情況——例如 報單號碼 被切成 報單 + 號碼 兩條 line,FindLabel 因為只比對單一 line 的文字而抓不到,改用本函式即可重組。

以 OneOcrClient 重組的 OcrResult.Rows 為基礎(同一水平帶的 line 併成一列、列內各欄位由左到右排序),在同一列內以滑動視窗找出「串接後可組成 AKeyword 的最短連續欄位區段」,回傳這些 box。取命中的最上列;列內取最短區段,長度相同取最左。

  • 比對前各 box 文字直接相接、忽略所有空白;lmStripParens 再去掉結尾的 "(數字)"。
  • lmContains (預設):區段包含關鍵字即可——適合關鍵字落在欄位中間,或 OCR 把「值」與「下一個標籤」黏在同一欄(例:274,601[調整金額]、-2,000[申報金額])的版面;lmExact / lmStripParens:整個區段串接後須等於關鍵字。
  • 同一列的欄位直接由左到右串接,不限制欄與欄之間的距離(同列即屬同一橫帶);靠「最短區段」優先避免抓到過寬的組合。
  • 若 Rows 未被填(TOneOcrClient.GroupRows 被關閉),內部會用 GroupLinesIntoRows 臨時重算,呼叫端不必特別處理。
  • 找不到時 Found=False、RowIndex=-1、Parts 為空。
FindKeywordBoxes vs FindLabel。FindLabel 要求一條 line 就完整等於 label;FindKeywordBoxes 允許 label 被拆成同列多個 box 再重組。label 一向是單一 line 時用 FindLabel 即可,只有遇到拆字才需要本函式。本函式只回 box 陣列,不負責沿方向取值。 
var
  M: TKeywordMatch;
  i: Integer;
begin
  M := Ex.FindKeywordBoxes('報單號碼');
  if M.Found then
  begin
    for i := 0 to High(M.Parts) do
      DrawQuad(Cv, M.Parts[i].Quad);   // 逐個畫出組成關鍵字的 box
    ShowMessage(M.Text);               // 重組後的文字;M.MergedQuad 為整體外框
  end;
end;
呼叫模式典型場景
FindLabel('報單號碼') + FindValueInDirection(Box, fdRight)整張圖中唯一存在的標籤,值在右側。
FindLabel('金額', TRegionRect.Make(...)) + FindValueInDirection(Box, fdRight)多個同名標籤,以區域區分後再取右側值。
FindLabel('總計') + FindValueInDirection(Box, fdDown)表頭欄位,值在下方同欄。
FindValueInDirection(Box, fdDown) + FindNextInDirection(prev, fdDown) × N表格同一欄連續抓多個值;fdDown 以中心 dy 排序,對欄高不齊比邊到邊距離更穩定。
呼叫 FindValueInDirection 後 TRegEx.IsMatch(Box.Value, '^\d+\.\d{2,4}$')需格式驗證時,呼叫端自行套 regex。
FindKeywordBoxes('報單號碼')label 被 OCR 拆成同列多個 box,需重組關鍵字並取得組成它的 box 陣列。

五、區域(TRegionRect)用法

區域使用原圖像素座標,與 OcrResult.Lines[i].Quad 同一座標系(若有開 PaddingPixels 已自動還原)。

// 整張圖
TRegionRect.Whole;

// 自訂矩形,例:左上 (100,200) 到右下 (800,400)
TRegionRect.Make(100, 200, 800, 400);

// 工具方法
R.IsWhole;                  // 是否為整張圖
R.ContainsPoint(X, Y);      // 點是否落入區域
R.ContainsCenterOf(Quad);   // Quad 的幾何中心是否落入區域
判定基準是「中心點」,不是「Quad 是否完全在內」。因此即便 OCR 框略微超出區域,只要中心點在內就會被認列。畫區域時可放寬幾個像素,容忍 OCR 框抖動。

六、標籤比對模式

模式行為適用情境
lmExact 完全相同(忽略大小寫)。 已知 label 文字無變化,例如英文標籤。
lmContains 子字串包含(忽略大小寫)。 label 可能有空白或不確定字尾。注意:容易誤命中,優先用區域限制範圍。
lmStripParens (預設) 先去掉結尾的 "(數字)" 再 Exact 比對。例:OCR 行 "報單號碼(3)",以 "報單號碼" 即可命中。 台灣制式表單最常見:欄位文字後接編號 (3)、(11) 之類。

七、方向(TFieldDir)用法

方向對齊條件主軸 / 距離典型情境
fdRight同列(Y 重疊 ≥ RowTolerance)X 軸主軸邊到邊距離,候選左緣 ≥ label 右緣 − slack「LABEL  VALUE」橫式表單(最常見)。
fdLeft同列(Y 重疊 ≥ RowTolerance)X 軸主軸邊到邊距離,候選右緣 ≤ label 左緣 + slacklabel 在欄位右邊;或反向確認用。
fdDown同欄(X 重疊 ≥ RowTolerance)中心 dy(候選中心 Y − label 中心 Y);方向以「候選中心 Y > label 中心 Y」判定表頭欄位,值列在下方;表格欄。中心 dy + 中心向下對欄高不齊與 OCR 框微重疊都較穩定。
fdUp同欄(X 重疊 ≥ RowTolerance)Y 軸主軸邊到邊距離;方向以「候選中心 Y < label 中心 Y」判定單位、註腳寫在上方時。
對同一個 label 試多個方向很便宜——FindLabel 只跑一次,之後 FindValueInDirection 只在記憶體裡的 Lines 線性掃描。可先試 fdRight,失敗再 fdDown,做 fallback。

九、屬性設定

屬性預設值說明
RowTolerance 0.5 同列/同欄判定門檻。兩 Quad 主軸區間重疊比例 ≥ 此值視為對齊。值越大越嚴格;若標籤與值高度差異大可降到 0.3,但會增加誤判風險。

Loaded (唯讀) False LoadImage 是否成功。False 時所有 Find 都會直接回未命中。
LastError (唯讀) '' LoadImage 失敗時的錯誤訊息(可能來自 TOcrFileResult.Message 或例外文字)。

十、錯誤與邊界情況

情境回傳行為
LoadImage 尚未被呼叫,或回 False所有 FindLabel / FindValueInDirection / FindNextInDirection 都回 Found=False 的空 TFieldBox。
區域內找不到符合的 labelFindLabel 回 Found=False,其餘欄位為空。
對 Found=False 的 Box 呼叫 FindValueInDirection 或 FindNextInDirection回傳全新空 Box(Found=False,LabelLineIdx=-1),不執行搜尋。
對 ValueLineIdx<0 的 Box 呼叫 FindNextInDirection沒有可用的下一輪錨點,直接回空 Box。
找到 label 但該方向無對齊候選Found=False,但 LabelText/LabelQuad/LabelLineIdx 已填(LabelLineIdx >= 0),Value=''、ValueLineIdx=-1。可保留 Box 試別的方向。
多個 label 命中取「最上、最左」者(以 Quad 上緣、左緣排序)。
多個候選依排序鍵由近到遠排序(fdRight/fdLeft/fdUp 用主軸邊到邊距離,fdDown 用中心 dy),取最近者。
距離超過label 另一軸尺寸 × 50排除,避免跨欄/跨列誤抓(以主軸邊到邊距離量,fdDown 亦同)。
FindKeywordBoxes 任何列都拼不出關鍵字回 Found=False、RowIndex=-1、Parts 為空。

十一、列舉並繪製到 VCL TImage

OCR 完成後,每一條被偵測到的文字行都存在 OcrResult.Lines[i](TOcrLine)裡,內含 Text 與 Quad(TOcrQuad = 四角 TOcrPoint,原圖像素座標)。每一條 Line 就是一個 TFieldBox 候選——FindLabel / FindValueInDirection 抽出的 LabelQuad / ValueQuad 都來自這個陣列。要在 VCL TImage 上把全部框畫出來,直接遍歷 OcrResult.Lines 即可。

畫在 Picture.Bitmap.Canvas,不要畫在 Image.Canvas。畫在 Picture.Bitmap.Canvas 上的線條會跟著影像一起儲存,並由 TImage 的 Stretch / Proportional 自動縮放;畫在 Image.Canvas 上的線會在 Repaint(切換視窗、最小化還原)時被擦掉。

1. 把原圖載進 TImage

uses
  Graphics, JPEG, OneOcrClient, OcrFieldExtractor;

procedure TForm1.LoadAndShow(const APath: string);
var
  Jpg: TJPEGImage;
begin
  Jpg := TJPEGImage.Create;
  try
    Jpg.LoadFromFile(APath);
    Image1.Picture.Bitmap.Assign(Jpg);   // 取得可寫的 Picture.Bitmap.Canvas
  finally
    Jpg.Free;
  end;
end;

2. 把 TOcrQuad 畫成多邊形的小工具

TOcrQuad 是 array[0..3] of TOcrPoint,TOcrPoint 的 X/Y 是 Single;TCanvas.Polygon 收 array of TPoint,要先 Round。

procedure DrawQuad(ACanvas: TCanvas; const AQuad: TOcrQuad);
var
  Pts: array[0..3] of TPoint;
  i: Integer;
begin
  for i := 0 to 3 do
    Pts[i] := Point(Round(AQuad[i].X), Round(AQuad[i].Y));
  ACanvas.Polygon(Pts);
end;

3. 列舉 OcrResult.Lines,把所有框畫到 TImage 上

procedure TForm1.DrawAllBoxes(Ex: TOcrFieldExtractor);
var
  i: Integer;
  Cv: TCanvas;
begin
  if not Ex.Loaded then Exit;

  Cv := Image1.Picture.Bitmap.Canvas;
  Cv.Brush.Style := bsClear;            // 只畫邊框,不填底色蓋住文字
  Cv.Pen.Color   := clRed;
  Cv.Pen.Width   := 2;
  Cv.Font.Color  := clBlue;
  Cv.Font.Size   := 9;

  for i := 0 to High(Ex.OcrResult.Lines) do
  begin
    DrawQuad(Cv, Ex.OcrResult.Lines[i].Quad);
    // 行號標在左上角(可選);文字直接落在影像座標上
    Cv.TextOut(
      Round(Ex.OcrResult.Lines[i].Quad[0].X),
      Round(Ex.OcrResult.Lines[i].Quad[0].Y) - 14,
      Format('#%d', [i]));
  end;

  Image1.Invalidate;                    // 觸發重繪
end;

4. 進階:用不同顏色標 Label 與 Value

有時想視覺化「目前抽到的欄位」——把 FindLabel + FindValueInDirection 拿到的 TFieldBox 用兩種顏色畫出來,即可一眼看出配對對不對。

procedure TForm1.HighlightField(const ABox: TFieldBox);
var
  Cv: TCanvas;
begin
  if not ABox.Found then Exit;

  Cv := Image1.Picture.Bitmap.Canvas;
  Cv.Brush.Style := bsClear;
  Cv.Pen.Width   := 3;

  Cv.Pen.Color := clLime;               // label = 綠
  DrawQuad(Cv, ABox.LabelQuad);

  if ABox.ValueLineIdx >= 0 then
  begin
    Cv.Pen.Color := clYellow;           // value = 黃
    DrawQuad(Cv, ABox.ValueQuad);
  end;

  Image1.Invalidate;
end;

// 用法
var
  Box, ValueBox: TFieldBox;
begin
  Box := Ex.FindLabel('報單號碼');
  ValueBox := Ex.FindValueInDirection(Box, fdRight);
  HighlightField(ValueBox);             // 同時畫出 label 與 value 的框
end;

5. 串完整流程(載圖 → OCR → 全部畫框 → 高亮欄位)

procedure TForm1.btnRunClick(Sender: TObject);
var
  Ex: TOcrFieldExtractor;
  Box, ValueBox: TFieldBox;
begin
  Ex := TOcrFieldExtractor.Create;
  try
    Ex.PaddingPixels := 25;
    if not Ex.LoadImage('C:\1.jpg') then
      raise Exception.Create(Ex.LastError);

    LoadAndShow(Ex.OcrResult.ImagePath);   // 先把原圖放進 Image1
    DrawAllBoxes(Ex);                      // 紅框:全部 OCR 行

    Box      := Ex.FindLabel('報單號碼');
    ValueBox := Ex.FindValueInDirection(Box, fdRight);
    HighlightField(ValueBox);              // 綠/黃框:命中的 label + value
  finally
    Ex.Free;
  end;
end;
座標系小提醒。OcrResult.Lines[i].Quad 用原圖像素座標,即使呼叫端開了 PaddingPixels,座標也已經由 OneOcrClient 自動還原回原圖(NormalizePaddedResult)。所以畫到「以原圖建出的 Picture.Bitmap」上不需要做任何縮放;若 Image1.Stretch=True 或 Proportional=True,顯示時 VCL 會自己處理。
不要邊跑 OCR 邊畫到 Image1.Canvas。TImage.Canvas 的內容會在控件下次 Paint 時被擦掉(切換視窗、滾動、改 Visible 都會觸發)。如果只是除錯 hover 效果,可以畫到 Image1.Canvas;但要持久顯示請畫到 Image1.Picture.Bitmap.Canvas。
不想動 VCL 也行。若只想看一張標註好的成品圖,LoadImage(APath, True) 會直接在原圖旁產生 <image>_J.jpg(由 xAISDK1.exe 標註),不需要自己畫。VCL 繪製的價值在於互動(滑鼠 hover 高亮、點選某行查值),而非靜態輸出。

十二、注意事項

  • TOcrFieldExtractor 不執行緒安全。若要平行處理多張圖,請每條 worker 各自 Create/LoadImage(底層 TOneOcrClient 也是同樣的限制)。
  • 一個實例只快取一張圖。要換圖時直接再呼叫 LoadImage 即可,內部會覆蓋舊結果。
  • 呼叫端可隨時讀 OcrResult.Lines 自行做更複雜的版面分析,此單元提供的 Find 只是常用情境的便利包裝。
  • Confidence 取的是「值那一行」各 word 的算術平均;若行內僅一個 word,即等於該 word 的 confidence。可用此值做品質門檻,例如 < 0.6 時提示人工檢核。
  • RowTolerance 0.5 對多數中文表單適用;遇到 label 字級顯著大於 value(常見於分節標題)時可往下調。
  • 若整張圖搜尋會撞到同名 label,優先用 TRegionRect 縮小範圍,而不是改用 lmContains。lmContains 容易把 "金額" 誤命中到 "金額合計"。
  • 需要格式驗證時,在呼叫 FindValueInDirection 後自行套 TRegEx.IsMatch(Box.Value, '...')。本單元刻意不混入 regex,讓抽取與驗證職責分離。
